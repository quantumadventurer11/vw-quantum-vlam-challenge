Methodology & Limitations
  • OpenVLA-7B was trained on Open X-Embodiment (bridge_dataset subset) — not on automotive or EV assembly tasks
  • Compression method: MPS/MPO SVD sweep via torch.linalg.svd, n_sites=4, bond dims χ∈{16,32,64,128}, 7 target weight types per LLM layer
  • Phase 3 L1 shift values are estimated using a log-χ scaling model anchored to CompactifAI results applied to Phase 2 Frobenius reconstruction error — not measured by running compressed model forward passes
  • Inference timing: Phase 1 baseline only (2095.6 ms ± 1.2 ms, 3 seeds × 50 episodes on lerobot/toto). Phase 3 timing was not separately measured for any χ variant
  • 3D visualizations (demo_3d.html, demo_factory.html) are schematic kinematic simulations driven by IK + synthetic drift arrays — not physics-based and not driven by real model outputs
  • OpenVLA-7B weights are released under LLaMA-2 Community License (non-commercial research only); code is MIT
  • Evaluation proxy: mean L1 action-prediction error on held-out lerobot/toto episodes (Open X-Embodiment bridge_dataset substitute); n=3 seeds × 50 episodes each
  • Phase 3 GPU eval failed due to Kaggle weekly quota exhaustion (30 h/week limit reached); results will be updated when quota resets (~2026-07-18)
